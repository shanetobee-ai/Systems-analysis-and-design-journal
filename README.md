# Systems-analysis-and-design-journal
Assignment repository
The DriverPass project was created for Liam, the owner of DriverPass, he wanted a web-based system that would help customers better prepare for their DMV driving exams. The system needed to support online classes, practice tests, account management, package selection, driving lesson reservations, progress tracking, and activity reporting. I believe I did well identifying the major areas needing improvement and connecting it to the system design. For example, customers needed access to accounts and lesson management.

If I could revise one part of my work, I would improve the use case diagram by making the relationships between the larger use cases and smaller supporting actions clearer. Some items, such as managing accounts and managing activity reports, work well as major use cases, while smaller actions like entering information or processing payment could be shown more clearly as supporting steps. I interpreted the user’s needs by reviewing what DriverPass wanted the system to accomplish and applying my findings into the diagrams. Considering user needs is important because software should make business problems easier to handle. In the future, I would approach software design by first gathering requirements, identifying the users, organizing the use cases, creating diagrams, and then reviewing the design.
